Message

消息是 LangChain 中模型上下文的基本单元。它们代表模型的输入和输出，携带与 LLM 交互时表示对话状态所需的内容和元数据。

消息是包含以下内容的对象：
角色 - 标识消息类型（例如 system、user）
内容 - 表示消息的实际内容（如文本、图像、音频、文档等）
元数据 - 可选字段，如响应信息、消息 ID 和 token 使用情况
LangChain 提供了一种适用于所有模型提供商的标准消息类型，确保无论调用哪个模型，行为都保持一致。

基本用法

使用消息最简单的方法是创建消息对象并在调用模型时将其传递给模型。

from langchain.messages import HumanMessage, AIMessage, SystemMessage
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI

load_dotenv()

# 使用通义千问模型
model = ChatOpenAI(
    model="qwen3.7-plus",
    api_key=os.getenv("OPENAI_API_KEY"),
    base_url=os.getenv("OPENAI_BASE_URL"),
)

# 系统消息 - 设定助手角色
system_msg = SystemMessage("你是一个乐于助人的AI助手。")

# 用户消息
human_msg = HumanMessage("你好，请介绍一下你自己")

# 构建消息列表
messages = [system_msg, human_msg]

# 调用模型
print("="*60)
print("通义千问模型测试")
print("="*60)
print(f"\n💬 系统: {system_msg.content}")
print(f"👤 用户: {human_msg.content}")

response = model.invoke(messages)

print(f"\n🤖 AI 回复:")
print(response.content)
print(f"\n📊 消息类型: {type(response).__name__}")

📋 系统指令: 你是一位乐于助人的编程助手。
👤 用户问题: 如何创建一个 REST API？

🤖 AI 回复:
创建一个 REST API 是一个系统性的过程，涉及**设计、编码、测试和部署**。为了让你清晰地理解，我将把这个过程分为几个关键步骤，并提供一个使用 **Node.js + Express** 的实战示例（这是目前最流行且易于上手的技术栈之一）。

---

### 第一步：理解 REST API 的核心设计原则

在写代码之前，必须先设计好 API 的结构。REST（表述性状态转移）的核心在于将数据视为**资源（Resources）**，并通过 **HTTP 方法**来操作它们。

1. **资源命名（URL 设计）**：
   * 使用**名词**，且通常使用**复数**形式。
   * 错误示例：`/getUser`, `/addNewUser`, `/user`
   * 正确示例：`/users`, `/users/123`, `/articles`
2. **HTTP 方法（动作）**：
   * **GET**：获取资源（读）
   * **POST**：创建新资源（增）
   * **PUT**：完整更新资源（改）
   * **PATCH**：部分更新资源（改）
   * **DELETE**：删除资源（删）
3. **HTTP 状态码**：
   * `200 OK`：请求成功。
   * `201 Created`：资源创建成功（通常用于 POST）。
   * `400 Bad Request`：客户端请求参数错误。
   * `401 Unauthorized`：未授权（需要登录）。
   * `404 Not Found`：资源不存在。
   * `500 Internal Server Error`：服务器内部错误。

---

### 第二步：选择技术栈

你可以根据团队熟悉度或项目需求选择后端语言：
* **Node.js**: Express, NestJS, Fastify
* **Python**: FastAPI, Django REST Framework, Flask
* **Java**: Spring Boot
* **Go**: Gin, Echo
* **C#**: ASP.NET Core Web API

*👉 下面的实战示例我们将使用 **Node.js + Express**。*

---

### 第三步：实战演练（Node.js + Express）

我们将创建一个管理“用户（Users）”的简单 CRUD API。

#### 1. 初始化项目
打开终端，运行以下命令：
```bash
mkdir rest-api-demo
cd rest-api-demo
npm init -y
npm install express
```

#### 2. 编写 API 代码
创建一个名为 `server.js` 的文件，并写入以下代码：

```javascript
const express = require('express');
const app = express();
const PORT = 3000;

// 中间件：解析 JSON 格式的请求体
app.use(express.json());

// 模拟数据库（实际项目中应使用 MySQL, MongoDB 等）
let users = [
    { id: 1, name: '张三', email: 'zhangsan@example.com' },
    { id: 2, name: '李四', email: 'lisi@example.com' }
];

// ================= CRUD 路由 =================

// 1. GET: 获取所有用户
app.get('/api/v1/users', (req, res) => {
    res.status(200).json(users);
});

// 2. GET: 根据 ID 获取单个用户
app.get('/api/v1/users/:id', (req, res) => {
    const user = users.find(u => u.id === parseInt(req.params.id));
    if (!user) return res.status(404).json({ message: '用户未找到' });
    res.status(200).json(user);
});

// 3. POST: 创建新用户
app.post('/api/v1/users', (req, res) => {
    const { name, email } = req.body;
    // 简单的输入验证
    if (!name || !email) {
        return res.status(400).json({ message: '姓名和邮箱是必填项' });
    }
    
    const newUser = {
        id: users.length > 0 ? users[users.length - 1].id + 1 : 1,
        name,
        email
    };
    users.push(newUser);
    res.status(201).json(newUser);
});

// 4. PUT: 更新用户信息
app.put('/api/v1/users/:id', (req, res) => {
    const user = users.find(u => u.id === parseInt(req.params.id));
    if (!user) return res.status(404).json({ message: '用户未找到' });

    const { name, email } = req.body;
    if (name) user.name = name;
    if (email) user.email = email;

    res.status(200).json(user);
});

// 5. DELETE: 删除用户
app.delete('/api/v1/users/:id', (req, res) => {
    const userIndex = users.findIndex(u => u.id === parseInt(req.params.id));
    if (userIndex === -1) return res.status(404).json({ message: '用户未找到' });

    users.splice(userIndex, 1);
    res.status(200).json({ message: '用户删除成功' });
});

// ================= 启动服务器 =================
app.listen(PORT, () => {
    console.log(`服务器已启动，监听端口 ${PORT}`);
});
```

#### 3. 运行并测试
在终端运行：`node server.js`

你可以使用 **Postman**、**Insomnia** 或 **cURL** 来测试：

* **获取所有用户 (GET)**:
  ```bash
  curl http://localhost:3000/api/v1/users
  ```
* **创建用户 (POST)**:
  ```bash
  curl -X POST http://localhost:3000/api/v1/users \
  -H "Content-Type: application/json" \
  -d '{"name": "王五", "email": "wangwu@example.com"}'
  ```

---

### 第四步：遵循 REST API 最佳实践

当你掌握了基础后，要写出企业级的 API，需要注意以下几点：

1. **版本控制**：在 URL 中加入版本号，如 `/api/v1/users`。当 API 发生重大改变时，可以平滑升级到 `/api/v2/`，而不会破坏老客户端。
2. **分页、过滤与排序**：当数据量大时，不要一次性返回所有数据。
   * 分页：`/users?page=2&limit=20`
   * 过滤：`/users?status=active&role=admin`
   * 排序：`/users?sort=-created_at` (按创建时间倒序)
3. **统一的错误响应格式**：
   不要有时返回字符串，有时返回对象。保持统一，例如：
   ```json
   {
     "error": {
       "code": 400,
       "message": "参数验证失败",
       "details": ["邮箱格式不正确"]
     }
   }
   ```
4. **安全性**：
   * **HTTPS**：生产环境必须使用 HTTPS 加密传输。
   * **认证与授权**：使用 JWT (JSON Web Tokens) 或 OAuth 2.0 保护需要登录的接口。
   * **速率限制 (Rate Limiting)**：防止恶意刷接口（如使用 `express-rate-limit`）。
   * **输入验证**：永远不要信任客户端输入，使用如 `Joi` 或 `Zod` 等库进行严格校验。
5. **连接真实数据库**：将示例中的内存数组替换为真实的数据库操作（使用 ORM 如 Prisma, Sequelize, 或 ODM 如 Mongoose）。

---

### 第五步：编写文档

一个没有文档的 API 是无法被前端或其他开发者顺利使用的。
* **Swagger / OpenAPI**：这是目前最流行的 API 文档标准。你可以使用工具自动生成可交互的 API 文档页面。
* **Postman Collections**：将你的请求集合导出，分享给团队。

### 总结

创建 REST API 的核心流程是：**设计资源与路由 -> 选择框架编写代码 -> 处理数据与数据库交互 -> 添加安全与验证机制 -> 编写文档并测试**。

你可以先把我提供的 Node.js 代码跑起来，感受一下 HTTP 请求和响应的过程，然后再尝试加入数据库（如 MongoDB 或 MySQL）和身份验证（JWT），这样你的技能就会得到实质性的提升！如果有特定语言或框架的需求，随时告诉我。

工具消息

对于支持工具调用的模型，AI 消息可以包含工具调用。工具消息用于将单个工具执行的结果传递回模型。工具可以直接生成 ToolMessage 对象。下面我们展示一个简单示例。请在工具指南中阅读更多内容。

import os
from dotenv import load_dotenv
from langchain_core.messages import SystemMessage, HumanMessage
from langchain_openai import ChatOpenAI

load_dotenv()

# 使用通义千问模型
model = ChatOpenAI(
    model="qwen3.7-plus",
    api_key=os.getenv("OPENAI_API_KEY"),
    base_url=os.getenv("OPENAI_BASE_URL"),
)
from langchain.messages import AIMessage
from langchain.messages import ToolMessage

# After a model makes a tool call
# (Here, we demonstrate manually creating the messages for brevity)
ai_message = AIMessage(
    content=[],
    tool_calls=[{
        "name": "get_weather",
        "args": {"location": "San Francisco"},
        "id": "call_123"
    }]
)

# Execute tool and create result message
weather_result = "Sunny, 72°F"
tool_message = ToolMessage(
    content=weather_result,
    tool_call_id="call_123"  # Must match the call ID
)

# Continue conversation
messages = [
    HumanMessage("What's the weather in San Francisco?"),
    ai_message,  # Model's tool call
    tool_message,  # Tool execution result
]
response = model.invoke(messages)  # Model processes the result
print(response)

消息内容

您可以将消息内容视为发送给模型的数据负载。消息具有一个松散类型的 content 属性，支持字符串和未类型化对象列表（例如字典）。这使得 LangChain 聊天模型能够直接支持提供商原生的结构，例如多模态内容和其他数据。此外，LangChain 为文本、推理、引用、多模态数据、服务器端工具调用和其他消息内容提供了专用内容类型。请参阅下面的内容块。LangChain 聊天模型在 content 属性中接受消息内容。这可能包含以下任意一种：

1.一个字符串

2.提供商原生格式的内容块列表

3.LangChain 的标准内容块列表
