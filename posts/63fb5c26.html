结构化输出

结构化输出允许智能体（Agents）以特定且可预测的格式返回数据。无需解析自然语言响应，您即可获得 JSON 对象、Pydantic 模型 或数据类形式的结构化数据，供应用程序直接使用。

LangChain 的 create_agent 可自动处理结构化输出。用户设置所需的结构化输出模式（Schema），当模型生成结构化数据时，该数据会被捕获、验证，并返回在智能体状态的 'structured_response' 键中。

响应格式

配置方式 说明 适用场景
ToolStrategy[Schema] 使用工具调用进行结构化输出 所有支持工具调用的模型（通用兼容）
ProviderStrategy[Schema] 使用提供者原生的结构化输出 OpenAI、Anthropic (Claude)、xAI (Grok) 等
type[Schema] (直接传 Pydantic 类) 自动选择最佳策略 推荐使用，LangChain 自动判断
None 未明确请求结构化输出 默认行为，返回普通文本

  1. ToolStrategy - 工具调用模式
    原理：将 Pydantic 模型转换为一个”虚拟工具”，让模型通过调用工具来输出结构化数据
    优点：
    ✅ 兼容性最好，适用于所有支持工具调用的模型
    ✅ 可靠性高，强制模型遵循 Schema
    缺点：
    ⚠️ 速度较慢（多一次工具调用）
    ⚠️ 通义千问需关闭思考模式才能使用

  2. .ProviderStrategy - 提供商原生模式
    原理：直接使用模型提供商的原生结构化输出能力（如 OpenAI 的 response_format 参数）
    优点：
    ✅ 速度快（直接输出，无需工具调用）
    ✅ 支持严格模式（strict mode），约束解码
    缺点：
    ❌ 仅支持特定模型（OpenAI GPT-4o+、Claude 3+、Grok 等）
    ❌ 通义千问、DeepSeek 等部分模型不支持

  3. AutoStrategy - 自动选择模式（推荐）
    原理：直接传入 Pydantic 模型类，LangChain 根据模型能力自动选择策略
    选择逻辑：

    如果模型支持原生结构化输出 → 使用 ProviderStrategy
  否则 → 降级到 ToolStrategy

    优点：
    ✅ 最简洁，代码最少
    ✅ 自动适配，无需手动判断
    ✅ 未来兼容性好

import os
from dotenv import load_dotenv
from langchain.chat_models import init_chat_model
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy, ProviderStrategy
from langchain.messages import HumanMessage
from pydantic import BaseModel, Field
from typing import List, Optional
import json

load_dotenv()

# 初始化模型 - 关闭思考模式以支持 tool_choice 参数
model = init_chat_model(
    model="qwen3.7-plus",
    model_provider="openai",
    api_key=os.getenv("OPENAI_API_KEY"),
    base_url=os.getenv("OPENAI_BASE_URL"),
    max_tokens=1024,
    # 关闭思考模式，避免 tool_choice 参数冲突
    extra_body={"enable_thinking": False},
)

print("结构化输出测试案例 - 使用 create_agent")

class PersonInfo(BaseModel):
    """个人信息的结构化数据"""
    name: str = Field(description="姓名")
    age: int = Field(description="年龄")
    email: str = Field(description="邮箱地址")
    phone: Optional[str] = Field(None, description="电话号码")

agent_auto = create_agent(
    model=model,
    response_format=PersonInfo,
    system_prompt="你是一个信息提取助手，从文本中提取个人信息。"
)

text_to_parse = "我叫张三，今年28岁，我的邮箱是zhangsan@example.com，电话是13800138000"

result_auto = agent_auto.invoke({
    "messages": [HumanMessage(content=f"请从以下文本中提取个人信息：{text_to_parse}")]
})

print(f"\n📝 原始文本: {text_to_parse}")
print(f"\n✅ 结构化结果:")
person_info = result_auto["structured_response"]
print(f"   姓名: {person_info.name}")
print(f"   年龄: {person_info.age}")
print(f"   邮箱: {person_info.email}")
print(f"   电话: {person_info.phone}")
print(f"\n📊 JSON 格式:")
print(json.dumps(person_info.model_dump(), indent=2, ensure_ascii=False))

class WeatherReport(BaseModel):
    """天气报告"""
    city: str = Field(description="城市名称")

class CourseInfo(BaseModel):

提供者策略

一些模型提供者通过其 API 原生支持结构化输出（例如 OpenAI、xAI (Grok)、Gemini、Anthropic (Claude)）。这是可用时最可靠的方法。要使用此策略，请配置 ProviderStrategy：

class ProviderStrategy(Generic[SchemaT]):
    schema: type[SchemaT]
    strict: bool | None = None

当您将模式类型直接传递给 create_agent.response_format 且模型支持原生结构化输出时，LangChain 会自动使用 ProviderStrategy

工具调用策略

对于不支持原生结构化输出的模型，LangChain 使用工具调用来实现相同的结果。这适用于所有支持工具调用的模型（大多数现代模型）。要使用此策略，请配置 ToolStrategy：

class ToolStrategy(Generic[SchemaT]):
    schema: type[SchemaT]
    tool_message_content: str | None
    handle_errors: Union[
        bool,
        str,
        type[Exception],
        tuple[type[Exception], ...],
        Callable[[Exception], str],
    ]

from dotenv import load_dotenv
from langchain.chat_models import init_chat_model
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy
from langchain.messages import HumanMessage
from pydantic import BaseModel, Field
from typing import Literal
import json

    model="qwen3.7-plus",
    model_provider="openai",
    api_key=os.getenv("OPENAI_API_KEY"),

class ProductReview(BaseModel):

错误处理策略

1.重试不是”重新调用”，而是基于完整对话历史的新一轮生成
2.模型能看到之前的错误，并据此修正输出
3.错误信息越详细，模型修正得越快
4.对话历史会不断增长，包含所有尝试记录
5.handle_errors=True 是启用重试的关键参数
6.这就是为什么自定义验证器的错误提示要写得详细——它在”教”模型如何修正！

您可以使用 handle_errors 参数自定义错误处理方式：

自定义错误消息:
ToolStrategy(
    schema=ProductRating,
    handle_errors="Please provide a valid rating between 1-5 and include a comment."
)
如果 handle_errors 是一个字符串，智能体将始终提示模型使用固定的工具消息进行重试。

class OrderData(BaseModel):
    """订单数据（带特定异常处理）"""
    order_id: str = Field(description="订单号")
    amount: float = Field(description="订单金额（必须是正数且不超过10000）")
    currency: str = Field(description="货币类型（只能是CNY、USD或EUR）")

    @field_validator('amount')
    @classmethod
    def validate_amount(cls, v):
        """
        验证金额：抛出 ValueError（会被重试）
        """
        if v <= 0:
            raise ValueError(
                f"❌ 业务验证失败: 金额 {v} 必须大于 0"
                f"\n   规则: 订单金额必须是正数"
                f"\n   当前值: {v}"
                f"\n   请修正为大于 0 的值（例如: {abs(v)}）"
            )

        if v > 10000:
            raise ValueError(
                f"❌ 业务验证失败: 金额 {v} 超过上限"
                f"\n   规则: 单笔订单不能超过 10000 元"
                f"\n   当前值: {v}"
                f"\n   请拆分为多个订单或降低金额到 10000 以下"
            )

        return v

    @field_validator('currency')
    @classmethod
    def validate_currency(cls, v):
        """
        验证货币类型：抛出 TypeError（不会被重试）
        """
        allowed_currencies = ["CNY", "USD", "EUR"]

        if v not in allowed_currencies:
            # 故意抛出 TypeError 而不是 ValueError
            # 这样就不会触发重试机制
            raise TypeError(
                f"🚫 严重错误: 不支持的货币类型 '{v}'"
                f"\n   允许的货币: {', '.join(allowed_currencies)}"
                f"\n   当前值: {v}"
                f"\n   这是系统配置问题，不会触发重试"
            )

        return v
