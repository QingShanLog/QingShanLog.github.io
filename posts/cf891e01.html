工具

工具扩展了智能体的能力，使其能够获取实时数据、执行代码、查询外部数据库并在现实世界中采取行动。

在底层，工具是可调用函数，具有明确定义的输入和输出，并传递给聊天模型。模型根据对话上下文决定何时调用工具以及提供哪些输入参数。

创建工具

创建工具最简单的方法是使用@tool装饰器。默认情况下，函数的文档字符串会成为工具的描述，帮助模型理解何时使用它。

from langchain.chat_models import init_chat_model
from langchain.tools import tool
import os

import dotenv
from langchain_core.messages import HumanMessage


@tool
def get_weather(location: str) -> str:
    """Get the weather for a given location."""
    return f"{location} is sunny"
dotenv.load_dotenv(override=True)
model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="deepseek",
    max_tokens=1024,
    streaming=True,
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
    verbose=True,
    model_kwargs={"temperature": 0.5},
)

invoke = model.invoke([HumanMessage(content="今天北京的天气如何")])
print(invoke)

print(get_weather.invoke("北京"))

类型提示是必需的，因为它们定义了工具的输入模式。文档字符串应信息丰富且简洁，以帮助模型理解工具的用途。

自定义工具名称

类型提示是必需的，因为它们定义了工具的输入模式。文档字符串应信息丰富且简洁，以帮助模型理解工具的用途。

高级模型定义

使用 Pydantic 模型或 JSON 模式定义复杂输入。

from pydantic import BaseModel, Field
from typing import Literal
from langchain.tools import tool

class WeatherInput(BaseModel):
    """Input for weather queries."""
    location: str = Field(description="City name or coordinates")
    units: Literal["celsius", "fahrenheit"] = Field(
        default="celsius",
        description="Temperature unit preference"
    )
    include_forecast: bool = Field(
        default=False,
        description="Include 5-day forecast"
    )

@tool(args_schema=WeatherInput)
def get_weather(location: str, units: str = "celsius", include_forecast: bool = False) -> str:
    """Get current weather and optional forecast."""
    temp = 22 if units == "celsius" else 72
    result = f"Current weather in {location}: {temp} degrees {units[0].upper()}"
    if include_forecast:
        result += "\nNext 5 days: Sunny"
    return result

访问上下文

当工具可以访问代理状态、运行时上下文和长期记忆时，它们的功能最强大。这使得工具能够做出上下文感知的决策、个性化响应并在对话中维护信息。

工具可以通过ToolRuntime参数访问运行时信息，该参数提供：

组件 描述 用例
状态 短期记忆 - 存在于当前对话期间的可变数据（消息、计数器、自定义字段） 访问对话历史，跟踪工具调用计数。
上下文 在调用时传递的不可变配置（用户 ID、会话信息） 根据用户身份个性化响应。
存储 长期记忆 - 跨对话持久存在的数据。 保存用户偏好，维护知识库。
流写入器 在工具执行期间发出实时更新。 显示长时间运行操作的进度。
执行信息 当前执行的身份和重试信息（线程 ID、运行 ID、尝试次数）。 访问线程/运行 ID，根据重试状态调整行为。
服务器信息 在 LangGraph Server 上运行时，服务器特定的元数据（助理 ID、图 ID、已认证用户）。 访问助理 ID、图 ID 或已认证用户信息。
配置 RunnableConfig用于执行 访问回调、标签和元数据。
工具调用 ID 当前工具调用的唯一标识符。 关联日志和模型调用的工具调用。

短期记忆(状态)

状态代表在对话期间存在的短期记忆。它包括消息历史记录以及您在图状态中定义的任何自定义字段。

访问状态

工具可以使用runtime.state访问当前对话状态。

import os

from langchain.tools import tool,ToolRuntime
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage

dotenv.load_dotenv(override=True)
model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="deepseek",
    max_tokens=1024,
    streaming=True,
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
    verbose=True,
    model_kwargs={"temperature": 0.5},
)
@tool
def get_last_user_message(runtime: ToolRuntime) -> str:
    """Get the most recent message from the user."""
    messages = runtime.state["messages"]

    # Find the last human message
    for message in reversed(messages):
        if isinstance(message, HumanMessage):
            return message.content

    return "No user messages found"

# Access custom state fields
@tool
def get_user_preference(
    pref_name: str,
    runtime: ToolRuntime
) -> str:
    """Get a user preference value."""
    preferences = runtime.state.get("user_preferences", {})
    return preferences.get(pref_name, "Not set")

更新状态

使用Command来更新智能体的状态。这对于需要更新自定义状态字段的工具非常有用。在更新中包含一个ToolMessage，以便模型可以看到工具调用的结果。

import os

import dotenv
from langchain.agents import AgentState
from langchain.chat_models import init_chat_model
from langchain.messages import ToolMessage, HumanMessage, AIMessage
from langchain.tools import ToolRuntime, tool
from langgraph.types import Command
dotenv.load_dotenv(override=True)
class CustomState(AgentState):
    user_name: str


@tool
def set_user_name(new_name: str, runtime: ToolRuntime[None, CustomState]) -> Command:
    """Set the user's name in the conversation state."""
    print("设置用户名:", new_name)
    return Command(
        update={
            "user_name": new_name,
            "messages": [
                ToolMessage(
                    content=f"User name set to {new_name}.",
                    tool_call_id=runtime.tool_call_id,
                )
            ],
        }
    )


# 额外的工具：获取用户名
@tool
def get_user_name(runtime: ToolRuntime[None, CustomState]) -> str:
    """Get the current user's name from state."""
    print("获取用户名:")
    return runtime.state.get("user_name", "Unknown")


# 额外的工具：更新多个状态字段
@tool
def update_user_profile(
    name: str = None,
    age: int = None,
    email: str = None,
    runtime: ToolRuntime[None, CustomState] = None
) -> Command:
    """Update user profile information."""
    updates = {"messages": []}
    
    if name:
        updates["user_name"] = name
    if age is not None:
        updates["user_age"] = age
    if email:
        updates["user_email"] = email
    
    # 添加工具响应消息
    updates["messages"].append(
        ToolMessage(
            content=f"Profile updated: name={name}, age={age}, email={email}",
            tool_call_id=runtime.tool_call_id,
        )
    )
    
    return Command(update=updates)


model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="deepseek",
    max_tokens=1024,
    streaming=True,
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
    verbose=True,
    model_kwargs={"temperature": 0.5},
)

上下文

上下文提供在调用时传递的不可变配置数据。它可用于用户 ID、会话详细信息或在对话期间不应更改的应用程序特定设置。通过runtime.context访问上下文：

from dataclasses import dataclass

import dotenv
import os
from langchain.chat_models import init_chat_model
from langchain.agents import create_agent
from langchain.tools import tool, ToolRuntime
dotenv.load_dotenv(override=True)

USER_DATABASE = {
    "user123": {
        "name": "Alice Johnson",
        "account_type": "Premium",
        "balance": 5000,
        "email": "alice@example.com"
    },
    "user456": {
        "name": "Bob Smith",
        "account_type": "Standard",
        "balance": 1200,
        "email": "bob@example.com"
    }
}

@dataclass
class UserContext:
    user_id: str

@tool
def get_account_info(runtime: ToolRuntime[UserContext]) -> str:
    """Get the current user's account information."""
    user_id = runtime.context.user_id

    if user_id in USER_DATABASE:
        user = USER_DATABASE[user_id]
        return f"Account holder: {user['name']}\nType: {user['account_type']}\nBalance: ${user['balance']}"
    return "User not found"

model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="deepseek",
    max_tokens=1024,
    streaming=True,
    temperature=0.5,
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
    verbose=True,
)
agent = create_agent(
    model,
    tools=[get_account_info],
    context_schema=UserContext,
    system_prompt="You are a financial assistant."
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "What's my current balance?"}]},
    context=UserContext(user_id="user123")
)

print("=" * 60)
print("测试结果")
print("=" * 60)
print("\nAgent 回复:")
print(result["messages"][-1].content)
print("\n" + "=" * 60)

长期记忆（存储）

BaseStore提供跨对话持久存在的存储。与状态（短期记忆）不同，保存到存储中的数据在未来的会话中仍然可用。通过runtime.store访问存储。存储使用命名空间/键模式来组织数据：

from typing import Any

from langchain.chat_models import init_chat_model
from langgraph.store.memory import InMemoryStore
from langchain.agents import create_agent
from langchain.tools import tool, ToolRuntime
import os

import dotenv
dotenv.load_dotenv(override=True)
model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="deepseek",
    max_tokens=1024,
    streaming=True,
    temperature=0.5,
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
    verbose=True,
)
# Access memory
@tool
def get_user_info(user_id: str, runtime: ToolRuntime) -> str:
    """Look up user info."""
    store = runtime.store
    user_info = store.get(("users",), user_id)
    print(f"{user_info}")
    return str(user_info.value) if user_info else "Unknown user"

# Update memory
@tool
def save_user_info(user_id: str, user_info: dict[str, Any], runtime: ToolRuntime) -> str:
    """Save user info."""
    store = runtime.store
    store.put(("users",), user_id, user_info)
    print("Successfully saved user info.")
    return "Successfully saved user info."

store = InMemoryStore()
agent = create_agent(
    model,
    tools=[get_user_info, save_user_info],
    store=store
)

# First session: save user info
agent.invoke({
    "messages": [{"role": "user", "content": "Save the following user: userid: abc123, name: Foo, age: 25, email: foo@langchain.dev"}]
})

# Second session: get user info
agent.invoke({
    "messages": [{"role": "user", "content": "Get user info for user with id 'abc123'"}]
})
# Here is the user info for user with ID "abc123":
# - Name: Foo
# - Age: 25
# - Email: foo@langchain.dev

流写入器

在执行期间从工具流式传输实时更新。这对于在长时间运行的操作期间向用户提供进度反馈非常有用。

对比性能

from typing import Any
from langchain.chat_models import init_chat_model
from langgraph.store.memory import InMemoryStore
from langchain.agents import create_agent
from langchain.tools import tool, ToolRuntime
from langgraph.types import Command
from langchain.messages import ToolMessage, HumanMessage
import os
import dotenv
import time

dotenv.load_dotenv(override=True)

model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="deepseek",
    max_tokens=1024,
    streaming=True,
    temperature=0.5,
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL"),
    verbose=True,
)

# ==================== 方式 1：不使用流写入器（同步）====================
@tool
def save_users_sync(users: list[dict[str, Any]], runtime: ToolRuntime) -> str:
    """批量保存用户信息（同步方式 - 阻塞）"""
    store = runtime.store
    start_time = time.time()

    saved_count = 0
    for user in users:
        user_id = user.get("id", f"user_{saved_count}")
        # 每次写入都要等待完成
        store.put(("users_sync",), user_id, user)
        saved_count += 1

        # 模拟处理延迟（实际场景中可能是网络 I/O、数据库操作等）
        time.sleep(0.1)  # 模拟每个用户保存需要 100ms

    elapsed = time.time() - start_time
    return f"✅ 同步方式：已保存 {saved_count} 个用户，耗时 {elapsed:.2f}秒"


# ==================== 方式 2：使用流写入器（异步）====================
@tool
def save_users_stream(users: list[dict[str, Any]], runtime: ToolRuntime) -> Command:
    """批量保存用户信息（流式方式 - 非阻塞）"""
    try:
        stream_writer = runtime.get_stream_writer()
    except AttributeError:
        # 如果环境不支持 stream_writer，降级为同步方式
        return save_users_sync.func(users, runtime)

    start_time = time.time()

    saved_count = 0
    for user in users:
        user_id = user.get("id", f"user_{saved_count}")
        # 流式写入，立即返回，后台继续处理
        stream_writer.write(
            ("users_stream",),
            user_id,
            {**user, "saved_at": time.time()}
        )
        saved_count += 1

        # 即使有延迟，也不会阻塞返回
        time.sleep(0.1)  # 模拟处理延迟

    elapsed = time.time() - start_time

    return Command(
        update={
            "messages": [ToolMessage(
                content=f"⚡ 流式方式：已提交 {saved_count} 个用户保存，耗时 {elapsed:.2f}秒（后台继续处理）",
                tool_call_id=runtime.tool_call_id
            )],
            "stream_save_count": saved_count,
            "stream_elapsed_time": elapsed
        }
    )

@tool
def get_saved_users_count(mode: str, runtime: ToolRuntime) -> str:
    """查询已保存的用户数量"""
    store = runtime.store

    if mode == "sync":
        namespace = ("users_sync",)
    elif mode == "stream":
        namespace = ("users_stream",)
    else:
        return "请指定模式：sync 或 stream"

    # 统计该命名空间下的所有用户
    count = 0
    try:
        for item in store.search(namespace):
            count += 1
    except Exception as e:
        return f"查询失败: {e}"

    return f"{mode} 模式下已保存 {count} 个用户"

@tool
def trace_call_chain(step_number: int, runtime: ToolRuntime) -> str:
    """追踪工具调用链路"""

    execution_info = runtime.execution_info

    call_info = {
        "step": step_number,
        "tool_name": getattr(execution_info, 'name', 'N/A'),
        "call_id": getattr(execution_info, 'call_id', 'N/A'),
        "parent_task_id": getattr(execution_info, 'parent_task_id', 'N/A'),
        "thread_id": getattr(execution_info, 'thread_id', 'N/A'),
        "timestamp": time.time()
    }

    # 将调用信息保存到状态中
    if "call_chain" not in runtime.state:
        runtime.state["call_chain"] = []

    runtime.state["call_chain"].append(call_info)

    return f"✅ 步骤 {step_number} 已记录到调用链"

@tool
def compare_executions(runtime: ToolRuntime) -> str:
    """比较历史执行信息"""

    call_chain = runtime.state.get("call_chain", [])

    if not call_chain:
        return "📭 暂无执行历史记录"

    summary_lines = [
        f"📊 **执行历史统计**:",
        "",
        f"总执行次数: {len(call_chain)}",
        "",
        "执行记录:",
    ]

    for i, call in enumerate(call_chain, 1):
        summary_lines.append(f"\n{i}. 步骤 {call['step']}")
        summary_lines.append(f"   工具: {call['tool_name']}")
        call_id = str(call['call_id'])
        summary_lines.append(f"   调用 ID: {call_id[:8] if len(call_id) > 8 else call_id}...")
        summary_lines.append(f"   时间: {time.strftime('%H:%M:%S', time.localtime(call['timestamp']))}")

    return "\n".join(summary_lines)

    """显示当前执行环境的服务器信息"""

    # ServerInfo 是对象，使用 getattr 访问属性
    info_lines = [
        "🖥️ **服务器/执行环境信息**:",
        "",
        f"• 主机名: {getattr(server_info, 'hostname', 'N/A')}",

        f"• 启动时间: {getattr(server_info, 'start_time', 'N/A')}",
    ]

    # 如果还有其他属性，可以动态获取
    try:
        # 尝试获取所有公共属性
        for attr in dir(server_info):
            if not attr.startswith('_'):
                value = getattr(server_info, attr, None)
                if value is not None and attr not in ['hostname', 'pid', 'thread_id', 'platform', 'python_version', 'start_time']:
                    info_lines.append(f"• {attr}: {value}")
    except Exception as e:
        info_lines.append(f"⚠️ 额外属性获取失败: {e}")

    return "\n".join(info_lines)

    if diagnostics:
        lines.append("\n最近诊断:")
        for i, diag in enumerate(diagnostics[-3:], 1):  # 显示最近 3 条
            lines.append(f"  {i}. {diag['timestamp']} - {diag['hostname']} (PID: {diag['pid']})")

    if performance:
        lines.append("\n性能趋势:")
        avg_memory = sum(p['memory_usage_mb'] for p in performance) / len(performance)
        lines.append(f"  平均内存使用: {avg_memory:.4f} MB")
        lines.append(f"  总记录数: {len(performance)}")

    return "\n".join(lines)


# ==================== 创建 Agent ====================
store = InMemoryStore()
checkpointer = MemorySaver()

agent = create_agent(
    model,
    tools=[
        show_server_info,
        diagnose_environment,
        monitor_performance,
        compare_server_history
    ],
    store=store,
    checkpointer=checkpointer
)

工具节点

ToolNode是一个预构建节点，用于在 LangGraph 工作流中执行工具。它自动处理并行工具执行、错误处理和状态注入。

基本用法

from langchain.tools import tool
from langgraph.prebuilt import ToolNode
from langgraph.graph import StateGraph, MessagesState, START, END

@tool
def search(query: str) -> str:
    """Search for information."""
    return f"Results for: {query}"

@tool
def calculator(expression: str) -> str:
    """Evaluate a math expression."""
    return str(eval(expression))

# Create the ToolNode with your tools
tool_node = ToolNode([search, calculator])

# Use in a graph
builder = StateGraph(MessagesState)
builder.add_node("tools", tool_node)
# ... add other nodes and edges

工具返回值

您可以为工具选择不同的返回值。

返回一个string以获取人类可读的结果。

返回一个object以获取模型应解析的结构化结果。

当您需要写入状态时，返回一个带有可选消息的Command。

返回字符串

from langchain.tools import tool


@tool
def get_weather(city: str) -> str:
    """Get weather for a city."""
    return f"It is currently sunny in {city}."

行为

返回值被转换为ToolMessage。

模型会看到该文本并决定下一步做什么。

除非模型或另一个工具稍后更改，否则智能体状态字段不会更改。

当结果是自然的人类可读文本时使用此方法。

返回对象

from langchain.tools import tool


@tool
def get_weather_data(city: str) -> dict:
    """Get structured weather data for a city."""
    return {
        "city": city,
        "temperature_c": 22,
        "conditions": "sunny",
    }

返回命令

当工具需要更新图状态时（例如，设置用户偏好或应用程序状态），返回一个Command。您可以返回一个带有或不带ToolMessage的Command。如果模型需要看到工具成功（例如，确认偏好更改），请在更新中包含一个ToolMessage，并使用runtime.tool_call_id作为tool_call_id参数。

from langchain.messages import ToolMessage
from langchain.tools import ToolRuntime, tool
from langgraph.types import Command


@tool
def set_language(language: str, runtime: ToolRuntime) -> Command:
    """Set the preferred response language."""
    return Command(
        update={
            "preferred_language": language,
            "messages": [
                ToolMessage(
                    content=f"Language set to {language}.",
                    tool_call_id=runtime.tool_call_id,
                )
            ],
        }
    )

行为

命令使用update更新状态。

更新后的状态可供同一运行中的后续步骤使用。

对可能被并行工具调用更新的字段使用 reducer。

当工具不仅返回数据，还修改智能体状态时使用此方法。

错误处理

from langgraph.prebuilt import ToolNode

# Default: catch invocation errors, re-raise execution errors
tool_node = ToolNode(tools)

# Catch all errors and return error message to LLM
tool_node = ToolNode(tools, handle_tool_errors=True)

# Custom error message
tool_node = ToolNode(tools, handle_tool_errors="Something went wrong, please try again.")

# Custom error handler
def handle_error(e: ValueError) -> str:
    return f"Invalid input: {e}"

tool_node = ToolNode(tools, handle_tool_errors=handle_error)

# Only catch specific exception types
tool_node = ToolNode(tools, handle_tool_errors=(ValueError, TypeError))

状态注入

工具可以通过ToolRuntime访问当前的图状态。

from langchain.tools import tool, ToolRuntime
from langgraph.prebuilt import ToolNode

@tool
def get_message_count(runtime: ToolRuntime) -> str:
    """Get the number of messages in the conversation."""
    messages = runtime.state["messages"]
    return f"There are {len(messages)} messages."

tool_node = ToolNode([get_message_count])
